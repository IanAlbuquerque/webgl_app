DrawingsHandler

DrawingsHandler

new DrawingsHandler()

A canvas drawings handler. It's responsible for creating all the buffers and methods related to drawing a WebGL scene.

Methods

beginDrawing(webgl_context, shader_program, primitive)

Begin buffering the vertices that will be drawn. It's expected to be followed by sequences of pushVertex2d or pushVertex3d, followed by endDrawing.

Parameters:
Name Type Description
webgl_context WebGLContext The WebGL canvas context that will be used for the drawing. This will be set as the current webgl context.
shader_program ShaderProgram The Shader Program that should be used. This will be set as the current shader program.
primitive WebGLPrimitive The primitive that should be used for the drawing. This will be set as the current drawing primitive.

drawVerticesColor2d(webgl_context, shader_program, primitive, vertices, colors, number_of_vertices)

Draws the given two dimesional vertices using the given WebGL primitive and a color per vertex. Note: The two dimensional vertices are actually treated as three dimensional vertices, with their third coordinate set to zero.

Name Type Description

endDrawing()

Draws the vertices stored in the vertices buffer with the methods pushVertex2d and pushVertex3d, using the current webgl context, current shader program and current drawing primitive set by the method beginDrawing.

pushVertex2d(x, y)

Adds a two dimensional (two entries) vertex to the vertices buffer. It's expected to be preceeded by beginDrawing. It's expected to be followed by endDrawing. Note: This two dimensional vertex is actually treated as a three dimensional vertex, with it's third coordinate set to zero. It doesn't conflict with pushVertex3d but it's mixed use should be avoided to improve code organization.

Parameters:
Name Type Description
x number The first coordinate of the two dimensional vertex to be buffered.
y number The second coordinate of the two dimensional vertex to be buffered.

pushVertex3d(x, y, z)

Adds a three dimensional (three entries) vertex to the vertices buffer. It's expected to be preceeded by beginDrawing. It's expected to be followed by endDrawing.

Parameters:
Name Type Description
x number The first coordinate of the three dimensional vertex to be buffered.
y number The second coordinate of the three dimensional vertex to be buffered.
z number The third coordinate of the three dimensional vertex to be buffered.
