MatricesHandler

MatricesHandler

new MatricesHandler()

A matrices operations handler. It's responsible for managing all the transformations and matrices-related operations of a WebGL application.

Members

private current_model_view_matrix :glMatrixArrayType

The current model view matrix. It's responsible for storing all the positioning transformations that should be applied to vertices being drawn.

private current_projection_matrix :glMatrixArrayType

The current projection matrix. It's responsible for storing the projection transformations that will be applied to the scene.

private model_view_matrix_stack :Array.<glMatrixArrayType>

The model view matrix stack. It's responsible for holding several model view matrices in order to deal with scenes that require storing different model view matrices.

Methods

mvLoadIdentity()

Sets the current model view matrix to the identity matrix. Overides any matrix values previously stored.

mvPopMatrix()

Deletes the current model view matrix and pops the last pushed matrix from the model view matrix stack, copying it as the new current model view matrix. Is expected to be used together with the method mvPushMatrix.

Throws:
Type
MatrixStackEmptyException

mvPushMatrix()

Pushes a copy of the current model view matrix into the model view matrix stack, preserving it for future uses. Is expected to be used together with the method mvPopMatrix

mvRotate(angle_in_radians, axe)

Multiplies (applies) a rotation matrix to the current model view matrix.

Parameters:
Name Type Description
angle_in_radians number Angle, in radians, of the rotation.
axe Array.<number> Axe to be used for the rotation. The matrix multiplied will be a rotation around this axe. Expects an array [x,y,z] with three entries.

mvScale(ratio)

Multiplies (applies) a scale matrix to the current model view matrix.

Parameters:
Name Type Description
ratio Array.<number> The [x,y,z] scale ratio of the scene. Expects an array with three entries. Use the third coordinate as one if drawing 2d scenes.

mvTranslate(vector)

Multiplies (applies) a translation matrix to the current model view matrix.

Parameters:
Name Type Description
vector Array.<number> The [x,y,z] vector to translate the scene. Expects an array with three entries. Use the third coordinate as zero if drawing 2d scenes.

pLoadIdentity()

Sets the current projection matrix as the identity matrix. Overides any matrix values previously stored.

pOrtho(left, right, bottom, top, near, far)

Sets the current projection matrix as a orthographic, given the necessary parameters. When using this kind of projection, further objects in the screne will appear as big as closer objects. This projection should be used for thow dimensional drawings.

Parameters:
Name Type Description
left number The left bound of the points of the scene.
right number The right bound of the points of the scene.
bottom number The bottom bound of the points of the scene.
top number The top bound of the points of the scene.
near number How close objects will still be rendered.
far number How far objects will still be rendered.

pSetPerspective(fovy, aspect, near, far)

Sets the current projection matrix as a three dimensional perspective projection, given the necessary parameters. When using this kind of projection, further objects in the screne will appear smaller than closer objects.

Parameters:
Name Type Description
fovy number The fovy aspect of the projection. (The angle of view).
aspect number The aspect ratio of the projection. (The ratio between the viewport width and height).
near number How close objects will still be rendered.
far number How far objects will still be rendered.

setMatrixUniforms(webgl_context, shader_program)

Loads the current model view matrix and the current projection matrix into the given WebGL canvas context and Shader Program.

Parameters:
Name Type Description
webgl_context WebGLContext The WebGL canvas context that will be used for loading the matrices.
shader_program ShaderProgram The Shader Program that will be used for loading the matrices.
