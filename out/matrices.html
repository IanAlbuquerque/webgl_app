webglut/matrices

webglut/matrices

The module that is responsible for the matrix operations in the canvas scene view.

Members

private inner mvMatrix :Matrix

The model view matrix. It's responsible for the positioning transformations in the scene.

private inner mvMatrixStack :Array.<Matrix>

The model view matrix stack. It's responsible for holding several model view matrices in order to deal with scenes that require multiple model view matrices.

private inner pMatrix :Matrix

The projection matrix. It's responsible for the projection transformations in the scene.

Methods

inner mvLoadIdentity()

Sets the current model view matrix as the identity matrix.

Example
Matrices.mvLoadIdentity();
// This cube will be free of model view transformations
drawCube();

inner mvPopMatrix()

Deletes the current model view matrix and recovers the last model view matrix from the stack, setting it as the current one. Is expected to be used together with the function mvPushMatrix.

Throws:
Type
Exception
Example
Matrices.mvTranslate([10,0,0]);
// This cube will be only Translated
drawCube();

Matrices.mvPushMatrix();
	Matrices.mvRotate(30,[0,0,1]);
	// This cube will be Rotated and Translated
	drawCube();
Matrices.mvPopMatrix();	

// This cube will be only Translated, exactly as the first one.
drawCube();

inner mvPushMatrix()

Pushes a copy of the current model view matrix into the stack, preserving it. Is expected to be used together with the function mvPopMatrix.

Example
Matrices.mvTranslate([10,0,0]);
// This cube will be only Translated
drawCube();

Matrices.mvPushMatrix();
	Matrices.mvRotate(30,[0,0,1]);
	// This cube will be Rotated and Translated
	drawCube();
Matrices.mvPopMatrix();	

// This cube will be only Translated, exactly as the first one.
drawCube();

inner mvRotate(angle, axis)

Applies a rotation to the current model view matrix.

Parameters:
Name Type Description
angle number Angle, in degrees, to rotate the scene.
axis Array.<number> Axis to be used for the rotation. Expects an array with three entries.
Example
Matrices.mvRotate(30,[0,1,0]);
// This cube will rotated by 30 degrees in the y axe.
drawCube();

inner mvScale(ratio)

Applies a scale to the current model view matrix.

Parameters:
Name Type Description
ratio Array.<number> The (x,y,z) scale ratio of the scene. Expects an array with three entries.
Example

// This cube will be not appear as a cube anymore. It will have it's dimentions scaled by the factor (10 times, 5 times, 1 time).

inner pOrtho(left, right, bottom, top, near, far)

Sets the current projection matrix as a ortographic projection. Usually used for two dimentional drawings. Further objects will appear as big as close objects.

Parameters:
Name Type Description
left number The left bound of the points of the scene.
right number The right bound of the points of the scene.
bottom number The bottom bound of the points of the scene.
top number The top bound of the points of the scene.
near number How close objects will be rendered.
far number How far objects will be rendered.
Example
GLModule.initialize(canvas);
ShadersModule.initialize();

gl.viewport(0, 0, gl.viewportWidth, gl.viewportHeight);
// Used to define the projection of a scene that draws in the XY plane in the interval given by the product [-10,10] x [-10,10]
Matrices.pOrtho(-10,10,-10,10,-1,1);

Events.setDisplayFunction(display);
Events.setLoopFunction(loop);
Events.initialize();

inner pSetPerspective(fovy, aspect, near, far)

Sets the current projection matrix as a three dimentional perspective projection, given the necessary parameters. Further objects will appear smaller than close objects.

Parameters:
Name Type Description
fovy number The fovy aspect of the projection. (The opening angle of view).
aspect number The aspect ratio of the projection. (The ratio between the view port width and height).
near number How close objects will be rendered.
far number How far objects will be rendered.
Example
GLModule.initialize(canvas);
ShadersModule.initialize();

gl.viewport(0, 0, gl.viewportWidth, gl.viewportHeight);
// Defines the perspective of the scene
Matrices.pSetPerspective(45, gl.viewportWidth / gl.viewportHeight, 0.1, 100.0);

Events.setDisplayFunction(display);
Events.setLoopFunction(loop);
Events.initialize();

inner setMatrixUniforms()

Loads the model view matrix and the projection matrix. into the scene related to the gl global object.

Example
// Implementation of the drawVerticesColor3d function
var drawVerticesColor3d = function(primitive,vertices,colors,numVertices)
{
	var vertexPositionBuffer = new Buffers.GLArrayBuffer(vertices,3,numVertices);
	var vertexColorBuffer = new Buffers.GLArrayBuffer(colors,4,numVertices);

	gl.bindBuffer(gl.ARRAY_BUFFER, vertexPositionBuffer.content);
	gl.vertexAttribPointer(shaderProgram.vertexPositionAttribute, vertexPositionBuffer.itemSize, gl.FLOAT, false, 0, 0);

	gl.bindBuffer(gl.ARRAY_BUFFER, vertexColorBuffer.content);
	gl.vertexAttribPointer(shaderProgram.vertexColorAttribute, vertexColorBuffer.itemSize, gl.FLOAT, false, 0, 0);

	// Loads the model view and projection matrices and draws the vertices given
	Matrices.setMatrixUniforms(gl,shaderProgram);
	gl.drawArrays(primitive, 0, numVertices);
}
